# Tailscale DERP 自动直连规则集

![Update Status](https://github.com/DawnAleax/tailscale-derp-rules/actions/workflows/update.yml/badge.svg)
![License](https://img.shields.io/github/license/DawnAleax/tailscale-derp-rules)

这个仓库通过 GitHub Actions 自动从 Tailscale 官方源抓取全球中继服务器（DERP）信息，并将其转换为 **OpenClash / Clash** 可用的 `rule-set` 规则集文件。

## 🚀 核心作用

Tailscale 组网时，如果双方无法建立 P2P 直接连接，流量会通过 DERP 服务器中继。
* **默认问题**：如果你的 OpenClash 将这些中继流量识别为“代理流量”，流量会经过你的代理节点中转，导致双重延迟。
* **解决方案**：通过此规则集，强迫所有 Tailscale DERP 节点走 **DIRECT（直连）**，显著降低组网延迟并提高稳定性。

## 📦 规则集地址

**YAML 格式 (Clash Rule-set):**
`https://raw.githubusercontent.com/DawnAleax/tailscale-derp-rules/main/tailscale_derp.yaml`

## 🛠 如何在 OpenClash 中使用

1.  **进入 OpenClash 菜单** -> **规则集设置**。
2.  点击 **添加**。
3.  按以下参数配置：
    * **名称**: `Tailscale_DERP`
    * **类型**: `HTTP`
    * **行为**: `classical`
    * **更新间隔**: `1440` (每天更新一次)
    * **规则集 URL**: 粘贴上面的“规则集地址”。
4.  **保存并应用**。
5.  在 **规则集分流预览** 或 **第三方规则管理** 中，将该规则集指向 **`DIRECT`** 策略。

## ⚙️ 自动化说明

* **数据来源**: [Tailscale 官方 DERP Map](https://login.tailscale.com/derpmap/default)
* **更新频率**: 每天北京时间上午 8:00 自动运行转换脚本。
* **包含内容**: 包含全球所有 DERP 节点的 `HostName` (域名)、`IPv4` 地址及 `IPv6` 地址。

## ⚠️ 注意事项

* 请确保你的 OpenClash 开启了 `自定义规则` 或正确配置了 `Rule Provider`。
* 建议在 Tailscale 客户端开启 `Direct Connection` 优化。
