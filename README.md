# Tailscale DERP 自动直连规则集

![Update Status](https://github.com/DawnAleax/tailscale-derp-rules/actions/workflows/update.yml/badge.svg)
![License](https://img.shields.io/github/license/DawnAleax/tailscale-derp-rules)

本仓库通过 GitHub Actions 自动从 Tailscale 官方 DERP Map 拉取全球中继服务器信息，并转换为 OpenClash / Clash Meta 可用的 Rule Provider 规则集。

---

## 🚀 核心作用

Tailscale 在无法建立 P2P 直连时，会通过 DERP 服务器进行中继。

如果不做处理：

- DERP 流量可能被 Clash 误判为代理流量
- 导致 “代理 → DERP → 代理” 双重转发
- 延迟显著增加

本规则集作用：

✔ 强制 DERP 流量 DIRECT 直连  
✔ 避免代理二次中转  
✔ 提升 Tailscale 组网稳定性

## 📁 规则文件结构

生成规则符合 Clash Rule Provider 标准：

```yaml
payload:
  - DOMAIN,derp1.tailscale.com
  - IP-CIDR,199.38.181.104/32
  - IP-CIDR6,2607:f740:f::bc/128
````

---

## 📊 OpenClash 规则类型说明

| 类型        | 说明                    | 是否推荐  |
| --------- | --------------------- | ----- |
| classical | DOMAIN + IP-CIDR 混合匹配 | ✔ 推荐  |
| ipcidr    | 仅 IP 匹配               | ❌ 不完整 |
| domain    | 仅域名匹配                 | ❌ 不完整 |

### 结论

DERP 必须使用 classical：

* DERP 同时存在域名 + IP 访问
* 仅 IP 会漏 DNS 解析流量
* 仅 DOMAIN 会漏直连 fallback IP

## 🔄 自动更新流程

GitHub Actions 执行流程：

1. 拉取 Tailscale DERP Map
2. 生成 rules（DOMAIN / IPv4 / IPv6）
3. 对比旧版本
4. 生成 CHANGELOG（节点增删）
5. 提交 commit
6. 创建 tag
7. 发布 Release
